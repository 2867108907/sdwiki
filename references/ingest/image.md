# 图像类知识摄取指南

LLM 直接读取图像文件，无需 OCR 或预处理脚本。

## 第1步：解读图像内容

仔细观察图像，提取：
- 文字内容：图中所有可见文字
- 视觉结构：流程图走向、树状层级、节点连接关系
- 核心信息：图像试图传递的结论或洞见
- 图表元素：坐标轴、图例、标注、箭头指向
- 上下文来源：图像出自哪里，用来解释什么问题

## 第2步：创建或更新知识文章

读取 `wiki/index.md` 检查相关知识文章是否存在：

### 如果知识文章尚不存在

向用户展示已有知识文章的路径，询问是否创建新文件：
    - 若用户要求创建新文件，则在 `wiki/` 下创建单一知识文章：
- **来源信息**（页面顶部）
  - 原始图像标题、出处作者、来源文档
  - 文档类型（流程图/信息图/思维导图/手绘笔记/照片扫描）
  - 核心摘要（2-3 句话概括：这张图解释了什么）

- **结构化转译**（主体部分）

  将视觉信息转为结构化知识。**核心原则**：
  - 讲透结构：还原逻辑关系，不遗漏节点连接
  - 讲透含义：解释每个模块、每个箭头的意义
  - 保留洞见：提取图像总结出的核心结论
  - 可编辑化：将视觉图转为 Mermaid 图表，便于后续维护

  **转译要求**：
  - 流程图 → 转为 Mermaid 流程图/状态图
  - 思维导图 → 转为 Mermaid 思维导图/树状图
  - 信息图 → 提取文字要点，表格化整理关键数据
  - 手绘笔记 → 整理作者思考脉络，补全跳跃逻辑
  - 扫描件 → 保留原图附件，文字转结构化内容

### 如果知识文章已存在：
- 向用户展示路径询问是否创建新文件
    - 若要求创建新文件，则创建单一知识文章同上述操作
    - 若要求更新
        - 用此图像的信息补充现有文章
        - 如果已有文本描述，可添加 Mermaid 图增强可视化
        - 将新来源添加到 frontmatter 字段
        - 添加发现的任何新连接
        - 如有必要更新 `confidence`（可视化验证 = 更高置信度）
        - 保留现有内容 —— 添加到它，不要替换它
