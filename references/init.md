# 初始化提示词

> 使用此命令在当前目录初始化一个新的 SDWiki 结构化知识库。

## 工作流程

### 第一步：检查当前目录状态

检查是否已存在知识库结构：
- 检查 `raw/` 目录
- 检查 `wiki/` 目录
- 检查 `wiki/index.md`
- 检查 `wiki/log.md`

### 第二步：根据状态处理

**如果已检测到完整的知识库结构：**
```
✅ 当前目录已是 SDWiki 知识库：
- raw/ 目录 ✓
- wiki/ 目录 ✓
- wiki/index.md ✓
- wiki/log.md ✓

可以直接使用：
- /sdwiki ingest <文档>  - 摄取新内容
- /sdwiki query <问题>  - 查询知识
```
直接结束，不需要再询问。

**如果检测到部分结构不完整或不存在：**
列出缺失什么，然后询问：
```
当前目录缺少 SDWiki 结构：
- 缺失 xxx

是否需要在当前目录创建 SDWiki 三层知识库结构？
```
等待用户确认后再创建。

### 第三步：确认后创建结构

用户确认创建后，执行：

1. 创建缺失的目录：
   - 创建 `raw/` 目录（原始资料存放，只读）
   - 创建 `wiki/` 目录（结构化知识存放，LLM 维护）

2. 创建缺失的初始文件：

`wiki/index.md` 模板：
```markdown
---
title: 知识库索引
type: index
created: {{today}}
updated: {{today}}
tags:
  - index
---

# 知识库索引

此文件由 SDWiki 自动维护，记录所有知识库内容。

## 按主题分类

### 示例主题

| 页面 | 类型 | 标签 | 摘要 |
|------|------|------|------|
| （新文章按此格式添加） | - | - | - |

## 统计

- 总文章数：0
- 概念文章：0
- 实体文章：0
- 数据文章：0
- 笔记文章：0
- 比较文章：0
- 综述文章：0
- 案例文章：0
```

`wiki/log.md` 模板：
```markdown
---
title: 操作日志
type: log
created: {{today}}
updated: {{today}}
tags:
  - log
---

# 知识库操作日志

最新操作记录在最上方。

| 日期 | 操作 | 详情 | 影响页面 |
|------|------|------|----------|
| {{today}} | init | 初始化知识库 | - |
```

### 第四步：输出结果

完成后告知用户：
```
✅ SDWiki 知识库初始化完成！

目录结构：
- ./raw/    - 原始资料层（存放输入文档，只读）
- ./wiki/   - 知识层（LLM 维护结构化知识）
- ./wiki/index.md  - 内容索引
- ./wiki/log.md    - 操作日志

开始使用：
- /sdwiki ingest path/to/file  - 摄取新内容
- /sdwiki query 问题           - 查询知识
```
