# 知识库结构规范

本文档定义了 Obsidian Vault 的结构、约定和工作流程。这是 LLM 维护知识库的指南。

---

## 架构概览

Vault 采用三层架构：

```
Vault/
├── raw/           # 原始资料层（只读）
├── wiki/          # 知识层（LLM 维护）
├── schema/        # 提示词工作流规范
└── output/        # 输出产物（编译报告/可视化/幻灯片）
```

> **说明**：SDWiki skill 本身已经**内置了 schema 层**（结构规范和工作流程定义），用户的知识库 vault 不需要 `schema/` 目录。

### 1. Raw 层（原始资料）
- 存放原始文档：文章、论文、书籍章节、网页剪藏
- 文件格式：.md, .pdf, .txt, .html, .docx, .xlsx 等
- **只读原则**：LLM 只读取，绝不修改
- 按来源类型可分子文件夹（可选）：
  - `Clippings/` - 网页剪切原始资料存放地
  - `extracted/` - 预处理脚本提取出的文本/CSV 存放地

### 2. Wiki 层（知识库）
- 存放 LLM 生成和维护的 Markdown 文件
- 文件类型定义：
  - **概念页**：解释特定概念、技术、方法、原理
  - **实体页**：人物、项目、公司、产品、工具、框架
  - **数据页**：数据集、统计数据、指标定义、数据结论、报表解读
  - **笔记页**：读书笔记、文章随记、个人思考、会议记录
  - **比较页**：多个概念/方案/技术的对比分析（表格形式）
  - **综述页**：跨多个来源的主题综合、知识汇总
  - **案例页**：实战案例、项目案例、代码示例讲解
- 特殊文件：
  - `index.md`：内容索引和目录
  - `log.md`：操作日志

### 3. Output 层（输出产物，可选）
- 整合报告源文件：`output/reports/` - Markdown 格式深度整合报告
- 专业 PDF 报告：`output/pdf/` - Typst 编译，高质量排版正式文档
- 可编辑 PPT 演示：`output/pptx/` - PowerPoint 原生 `.pptx` 格式，可任意修改
- 知识关系可视化：`output/visuals/` - 知识图谱可视化
- 产物由 LLM 生成，用于外部分享或展示，全部保存在 output/ 下

## 命名约定

### 文件命名
- 使用小写字母和连字符：`-`
- 英文优先（如果原文是英文）
- 中文可用（如果内容以中文为主）
- 避免空格，使用连字符替代

示例：
- `transformer-architecture.md`
- `attention-mechanism.md`
- `karpathy-llm-wiki-concept.md`

### Wiki 页面类型标识
在 frontmatter 中用 `type` 字段标识：

- `type: concept` - 概念页（解释概念、技术、方法）
- `type: entity` - 实体页（人物、项目、公司、产品、工具）
- `type: data` - 数据页（数据集、统计、指标、报表）
- `type: note` - 笔记页（随记、思考、记录）
- `type: comparison` - 比较页（多个方案/技术对比）
- `type: synthesis` - 综述页（跨来源主题综合）
- `type: case` - 案例页（实战案例、项目案例）
- `type: index` - 索引页
- `type: log` - 日志页

---

## Frontmatter 规范

每个 wiki 页面必须包含 YAML frontmatter：

```yaml
---
title: 页面标题
type: concept|entity|data|note|comparison|synthesis|case|index|log
created: YYYY-MM-DD
updated: YYYY-MM-DD
source: 原始来源（可选）
tags:
  - tag1
  - tag2
---
```

---

## 链接规范

### 内部链接（Wiki-links）
- 使用 Obsidian 的 Wiki-link 语法：`[[页面名称]]`
- 优先使用简洁的链接文本
- 可以添加别名：`[[页面名称|显示文本]]`

### 外部链接
- 使用标准 Markdown：`[显示文本](URL)`
- 优先使用 HTTPS

## 内容结构

### 概念页结构
1. **概述**：一句话定义
2. **核心要点**：3-5 个关键要点
3. **详细解释**：深入说明
4. **相关概念**：链接到其他页面
5. **参考资料**：来源链接

### 实体页结构（人物/项目/公司/产品）
1. **基本信息**：简介、背景
2. **关键贡献/特性**：主要成就或功能
3. **相关实体**：关联的人物、产品、项目
4. **时间线**（可选）：重要事件

### 数据页结构
1. **数据源信息**：标题、发布者、时间范围、获取地址
2. **数据定义**：字段含义、统计口径、度量单位
3. **核心发现**：关键指标、趋势结论、异常点
4. **相关分析**：关联关系、影响解读
5. **局限性**：数据可能存在的偏差

### 笔记页结构
1. **主题概述**：笔记的主题背景
2. **要点记录**：分点记录核心内容
3. **个人思考**：自己的理解、感悟、问题
4. **关联知识**：链接到知识库中相关概念

### 比较页结构
1. **对比背景**：为什么比较，对比的对象是什么
2. **对比维度**：表格形式展示各维度对比
3. **结论**：各自适用场景、推荐选择
4. **参考资料**：来源链接

### 综述页结构
1. **主题引言**：这个主题是什么，为什么重要
2. **发展脉络**：历史演进、主要流派
3. **核心概念梳理**：整合各来源的关键知识
4. **现状总结**：当前共识、开放问题
5. **参考来源**：列出所有参考文章

### 案例页结构
1. **案例背景**：项目背景、问题描述
2. **解决方案**：采用的方法、技术选型
3. **实施过程**：关键步骤、踩过的坑
4. **结果复盘**：成果、经验、教训
5. **相关概念**：链接用到的技术概念
